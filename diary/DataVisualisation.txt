The main story of my dashboard is helping a stakeholder understand where reimbursement pressure is highest and what cost should be expected for a drug in the next quarter. Instead of building a general research dashboard, I shifted the design toward a customer-focused decision dashboard. The central question became: if I select a drug, a state, and the next quarter, what cost per unit should I expect? This made the dashboard more practical and useful for decision-making. I included prediction widgets using ipywidgets so users can select a drug NDC and a state and directly estimate the next-quarter Medicaid cost per unit using the trained LightGBM model. I also included visuals for cost distribution, reimbursement trends over time, state comparisons, therapeutic class drivers, and supplier concentration. These charts help explain why some drugs or markets are more expensive than others. I removed the raw action table because it made the dashboard too crowded and less useful for stakeholders. The final design focuses on clear communication, quick decision support, and highlighting which drugs and states deserve attention first.
