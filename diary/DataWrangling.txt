I started cleaning by focusing on the variables directly related to reimbursement behavior because the goal of the project is to study pricing differences across states for the same drug. The first issue I fixed was invalid target values, especially rows where cost per unit would become zero, negative, or undefined because units reimbursed were zero or missing. I removed those rows since they would distort model training and pricing comparisons. I also handled suppressed rows carefully because low-volume reporting often creates misleading averages. I converted important columns like year, quarter, units reimbursed, prescriptions, and reimbursement amounts into proper numeric types so calculations would remain consistent. I standardized categorical columns such as state names, route names, and labeler names by removing spacing issues and inconsistent formatting. I defined uniqueness using state, quarter, and NDC level records and checked duplicates before keeping the final rows. I also removed obvious outliers in cost per unit using percentile trimming so extreme values would not dominate the model. Before cleaning, the dataset had noisy rows and inconsistent formatting; after cleaning, the dataset became stable and analysis-ready. One remaining risk is that removing extreme values may also remove true rare expensive drugs, so that decision may affect interpretation later. I documented these steps clearly so the final dataset can be reproduced and defended later.
