Feature engineering was one of the most important parts of this project because raw reimbursement values alone were not enough to explain pricing behavior. I created cost_per_unit using Medicaid amount reimbursed divided by units reimbursed because this became the main target variable for prediction. I also created units_per_prescription by dividing units reimbursed by number of prescriptions so the model could better capture utilization intensity instead of only total volume. I built FDA_ProductID by combining the labeler segment and product code from PRODUCTNDC so I could connect Medicaid utilization data with FDA reference product information. This allowed me to merge Therapeutic Class, EPC Class, and Mechanism of Action features from the FDA product file. I also included route name, labeler name, and marketing category because manufacturer behavior and formulation often influence reimbursement levels. I created is_expansion_state as a binary feature to capture policy differences across Medicaid expansion and non-expansion states. During feature selection, I removed weak identifiers and columns with high missingness that did not improve prediction quality. I kept features that had clear business meaning and direct relevance to reimbursement pricing. This made the final model more interpretable and stronger for prediction.
