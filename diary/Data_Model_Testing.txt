This week focused on making the LightGBM model portable and reproducible outside the original notebook. I saved the final trained model as lgbm_medicaid_model.pkl using joblib. I chose joblib instead of pickle because it works better for larger machine learning models, especially models like LightGBM that contain many trees and NumPy-based structures. It is faster and more reliable for saving and loading compared to pickle for this type of project.

To test deployment, I created a separate file called model_testing.py that loads the saved model and runs predictions independently. This helped verify that the model was not dependent on hidden notebook state or previously executed cells. The testing file checks if both the model file and df_model.csv exist, loads them, normalizes the column names, selects the required features, and runs predictions on sample rows.

One issue I faced was a ModuleNotFoundError for LightGBM while loading the .pkl file in another environment. This happened because LightGBM was not installed in that environment. After installing the package, the model loaded successfully and predictions worked correctly. This confirmed that the workflow is reproducible and the model can be deployed outside the training notebook.
